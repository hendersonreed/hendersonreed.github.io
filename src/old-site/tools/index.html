TL;DR

I live on the command line:

  • writing (code or essays): vim + pandoc (see below)
  • build tooling: make
  • languages: Python, Bash, C. I've also been really enjoying Haskell, and I've toyed around with Rust.
  • pandoc for generating pdfs and html from markdown.
  • I am currently running Fedora on a Dell XPS 13 laptop. I find that the hardware has pretty great linux support, and Fedora is a pretty good balance between "batteries-included" and customizability.

I am a big proponent of the "Unix as IDE" philosophy; I recommend you check out the canonical series if you haven't encountered it before

code manipulation

I do nearly all of my work in vim. I really recommend that, if you haven't already taken a side in the vim-emacs-ide war, you should take a look at vim. It's an extensible, flexible, and, once you get used to it, incredibly comfortable tool.

I have chosen to install a few plugins, using the vim-plug plugin manager:

  • ALE (an Asynchronous Linting Engine, paired with pylint, gcc, and shellcheck, a linter for bash scripts.)
  • vim.fugitive (Tim Pope's well known git wrapper, I use it for the diff and blame features.
  • rainbow parentheses (highlights nested parens in different colors.

If you aren't familiar with the quickfix list, I recommend you learn about it - it allows you to prepopulate an easily navigable list with search results and compilation errors.

Also, I have found that vim macros are an often overlooked and underrated feature.

Aside from vim, I do some work in PyCharm, which is a JetBrains product. I'm enjoying it, though it's not as comfy as vim yet. But the Git integration is awesome for looking at diffs of previous versions of code. This is especially useful if you are editing someone else's code and you accidentally broke it and need to see how it worked before you went and mucked it up. Not that I know from personal experience....

misc.

Occasionally I use tmux to setup multiple sessions. This is especially handy if you are trying to debug something on a remote server and you want multiple splits or tabs open.

I also use a few home-built tools, like the below bash functions for managing my notes: 

mdless() {  
  pandoc -s -f markdown -t man $1.md | groff -T utf8 -man | less  
}  
umedit() { 
  mkdir -p ~/documents/linux_stuff/knowledge; 
  vim ~/docs/linux_stuff/knowledge/$1.md; }  
um() { mdless ~/documents/linux_stuff/knowledge/"$1"; }  
umls() { ls ~/documents/linux_stuff/knowledge; }
